# Requirements: Editor (index)

The editor workflow requirements are split into one file per activity:

- [`editor-configure.md`](editor-configure.md) — scan layer selection, island review and exclusion, symmetry confirmation
- [`editor-overview.md`](editor-overview.md) — map name, version, authors
- [`editor-teams.md`](editor-teams.md) — team definitions, kits, spawn placement, spawn access rules
- [`editor-build-regions.md`](editor-build-regions.md) — max build height, build area declaration, boundary enforcement, lockdowns and physics
- [`editor-objectives.md`](editor-objectives.md) — wool objective placement, wool room access rules, wool availability check
- [`editor-filters.md`](editor-filters.md) — rule review, block renewal, resistance reset, jump pads, lend-kit zones, map boundary, time-gated features
- [`editor-regions.md`](editor-regions.md) — region inventory, unassigned region inspection, symmetry violation review

For the activity order, status indicator behaviour, notification system, export rules, and entry point descriptions, see `plans/editor-vision.md`.

For the source domain model and authoring narrative, see `docs/map-data-model.md` and `docs/ctw-map-pamphlet.md`.
